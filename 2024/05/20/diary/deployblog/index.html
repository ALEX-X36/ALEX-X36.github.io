重新部署我的博客

  • 2024-05-20
  • 作者 MoChen
  • 1225 字
  • 次阅读

本文最后编辑于 前，其中的内容可能需要更新。

今天在修改博客风格的时候，发现 GitHub 上部署的主题没有同步改变，所以打算换个部署方式，不知道可不可行。hexo教程里面关于部署分支和写作分支的提示不太明白。

For 使用 Git 管理站点目录的用户

由于 Hexo 的部署默认使用分支 master，所以如果你同时正在使用 Git 管理你的站点目录，你应当注意你的部署分支应当不同于写作分支。
一个好的实践是将站点目录和 Pages 分别存放在两个不同的 Git 仓库中，可以有效避免相互覆盖。
Hexo 在部署你的站点生成的文件时并不会更新你的站点目录。因此你应该手动提交并推送你的写作分支。

放两个仓库的话，站点一个、Pages一个，但是 Pages 指的是生成的静态文件还是哪个？？应该是静态文件。但我的写作分支手动提交的话，本地要先 hexo deploy 一下站点，然后用 git 推送吗？？实践出真知还是试一试吧。

首先直接试一下只执行 deploy 根据他给的教程，应该是根据 _config.yml 里面的配置自动上传部署的。

  • 没反应，应该还要先执行 clean 在执行 deploy。
  • 可以了，但是 GitHub Pages 没有构建，改了 Source 变为 Deploy from a branch 才进行的构建。

后面的尝试暂且不弄了，估计要在 public 目录里面进行 git init 然后关联另一个用来部署博客的仓库。但说实话先 clean 再 deploy 更方便一些吧？除非博客体量非常大，那到时候要先生成静态文件然后推送。

emm要不试一下吧。

  • 首先使用 hexo generate 命令
  • 然后进入 public 目录下，进行 git 的初始化 git init
  • 关联远程仓库 git remote add origin 仓库地址
  • 然后尝试了下 git pull 根据结果来看，应该只 pull dp-pages 分支，即使用 git pull origin dp-pages 命令。
  • 将文件加入缓存 git add . ，然后提交 git commit -m "消息"
  • 最后推送 git push -u origin dp-pages

推不上去，还是使用 deploy 吧，估计要分仓库才行，只弄分支不可以。

解决了，先删除里面的文件，再 pull ，完整流程应该是

  • hexo clean && hexo deploy 初始部署一下。配置里面仓库作为项目仓库。
  • 创建空仓库，命名为用户名.git.io 做为博客网站仓库
  • 然后初始化、关联仓库、拉取、添加文件进缓存、提交、推送。

啊又尝试了一下总结下来就是：如果要分仓库那就在 public 里面提交。如果不分那后面新的内容就用 deploy，但是这里面有个清空文件然后再从 public 复制文件的操作，估计后面内容多了还是会慢，所以还是要分仓库吧？
